# Momo Study Agent - 架构与模块深度分析

本项目是一个高度工程化的自动化系统，旨在通过 Google Gemini AI 为墨墨背单词的生词提供深度“雅思级”解析。以下是对核心模块的设计初衷、关键特性及工程考虑的全面分析。

---

## 1. 配置中心层 (`config.py`)
**定位**：系统的“神经中枢”与环境配置中心。

### 关键特性
- **环境变量统一管理**：封装了对 `.env` 的读取，确保 `MOMO_TOKEN` 和 `GEMINI_API_KEY` 等敏感信息的集中引用。
- **路径编排**：动态计算项目根目录，自动创建 `data/` 目录，确保数据库和 Prompt 文件的路径在不同操作系统下均能透明访问。
- **运行时控制开关 (`DRY_RUN`)**：引入了全局安全开关。默认开启 `DRY_RUN`，防止在调试期间向 Maimemo 云端写入错误数据。

### 设计考量
- **零侵入性**：其它模块不直接调用 `os.getenv` 或 `load_dotenv`，而是通过 `from config import ...` 引用，极大降低了环境迁移的复杂性。

---

## 2. 墨墨 API 封装层 (`maimemo_api.py`)
**定位**：云端交互的“桥梁”，负责所有网络通信。

### 关键特性
- **全接口覆盖**：实现了查询单词、获取今日学习列表、同步释义、管理笔记等全量 OpenAPI 功能。
- **智能同步逻辑 (`sync_interpretation`)**：采用“先查后写”模式。如果单词下已有用户释义，则执行 `Update`（覆盖同步）；若无，则执行 `Create`。
- **内置频控**：根据墨墨官方频控建议，在底层请求中加入 `time.sleep(0.5)`，确保在大批量同步时不会被封禁。

### 设计考量
- **容错处理**：对 API 返回的业务状态码（success）进行严格检查，并提供详细的错误日志输出。

---

## 3. AI 交互引擎 (`gemini_client.py`)
**定位**：知识生成的核心，负责 AI 提示词编排与结果清洗。

### 关键特性
- **鲁棒的 JSON 提取器 (`_extract_json_array`)**：针对 Gemini 长输出时可能出现的“末尾幻觉乱码”问题，自研了**括号计数器算法**。通过精确匹配第一个合法闭合的 `]` 位置，彻底解决了 JSON 解析崩溃。
- **重试与退避机制**：实现了针对 `429 (Resource Exhausted)` 和 `503 (Unavailable)` 错误的指数退避重试，显著提升了在免费额度波动下的运行成功率。
- **Markdown 清洗**：自动剥离模型输出的 Markdown 代码块（```json ... ```）。

### 设计考量
- **指令解耦**：将复杂的 Prompt 逻辑存放在外部 `gem_prompt.md` 中，使得在不改动 Python 代码的情况下即可微调 AI 的分析深度。

---

## 4. 数据库管理层 (`db_manager.py`)
**定位**：持久化核心与“安全隔离仓”。

### 关键特性
- **环境隔离 (Database Isolation)**：支持动态传入 DB 路径。通过逻辑区分 `history.db`（正式库）和 `test.db`（测试库），确保实验不污染生产数据。
- **知识图谱存储**：定义了 `ai_word_notes` 表，支持存储包括“雅思考点”、“词义辨析”、“记忆助记”在内的 10 余个维度的深度解析数据。
- **处理历史追踪**：维护 `processed_words` 表，用于增量同步时的去重判定，避免重复消耗 AI 额度。

### 设计考量
- **Schema 同步性**：无论在测试还是正式环境下，程序共用同一套 Schema 定义，确保测试结果具有 100% 的生产参考价值。

---

## 5. 流程编排层 (`main.py`)
**定位**：任务调度者，将 API、AI 与 DB 逻辑串联。

### 关键特性
- **面向对象封装 (`StudyFlowManager`)**：将过程式的同步逻辑封装为类，更方便地共享 API 客户端实例并进行流程管理。
- **批量化处理模式**：支持按预设的 `BATCH_SIZE` 进行分批处理，在提效的同时保持 API 的稳定性。
- **增量同步过滤 (Incremental Sync)**：在同步开始前，预先从数据库对比“已处理列表”（`processed_words`），仅对真正的新生词发起 AI 请求。
- **成本与性能优化**：由于 AI 额度和响应速度有限，系统实现了“本地数据库优先”原则。如果本地已存在该单词的分析记录，则直接跳过 AI 处理环节，极大节省了 API 消耗并提升了运行速度。

### 设计考量
- **终端交互友好**：在脚本入口强制进行 UTF-8 重配置（针对 Windows），并采用精心设计的极简日志输出风格。

---

## 6. 知识核芯 (`gem_prompt.md`)
**定位**：定义了 AI 生成结果的“灵魂”。

### 关键逻辑
- **雅思针对性分析**：强制 AI 关注 `ielts_focus`（考点）和 `synonyms`（写作提分替换词）。
- **合并合并逻辑**：要求 AI 将功能相近的义项合并，并用 `[生僻]` 标注低频义项，最大化减轻记忆负担。

---

## 7. 核心工程准则 (`.antigravity_rules.md`)
**定位**：项目长期维护的“工程法典”。

### 核心法条
1. **数据安全**：任何写入操作必须先过测试环境。
2. **容错优先**：JSON 解析不能依赖正则，必须使用逻辑计数。
3. **隔离开发**：生产与测试物理隔离。
