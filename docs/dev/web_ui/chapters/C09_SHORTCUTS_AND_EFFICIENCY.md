# C09 快捷键与效率交互

状态：confirmed  
更新时间：2026-05-02

## 1. 章节目标

建立稳定、可学习、低误触的快捷键体系，提升高频操作效率。

## 2. 已确认决策

1. 快捷键总量：8 个核心快捷键
2. 提示方式：首次进入展示引导浮层
3. 执行类防误触：必须组合键（如 Ctrl+Enter）
4. 作用范围：全局生效，输入框内自动禁用危险键
5. 支持 `?` 快速显示快捷键速查面板
6. 不允许用户自定义快捷键（先保一致性）
7. 冲突优先级：系统/浏览器优先，应用自动降级
8. 记录快捷键使用埋点

## 3. 章节计划（可执行）

1. 交互层
   - 首次引导浮层
   - `?` 速查面板
2. 安全层
   - 执行类快捷键统一组合键
   - 输入框场景禁用危险键
3. 冲突层
   - 建立冲突检测与降级机制
4. 观测层
   - 记录快捷键埋点用于优化

## 4. 验收标准

1. 全站 8 个核心快捷键可用且行为一致。
2. 首次进入展示快捷键引导浮层。
3. `?` 能打开速查面板。
4. 执行类快捷键均为组合键且无误触。
5. 输入框内危险快捷键自动禁用。
6. 冲突场景优先系统/浏览器，应用降级不报错。
7. 快捷键埋点记录完整。

## 5. 风险与回退

1. 风险：快捷键学习成本。
   - 处置：首次引导 + 速查面板。
2. 风险：浏览器冲突。
   - 处置：应用降级并提示替代操作。
3. 风险：埋点噪音。
   - 处置：仅采集必要字段与匿名统计。
