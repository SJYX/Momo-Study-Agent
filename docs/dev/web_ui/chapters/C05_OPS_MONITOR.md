# C05 Ops Monitor 监控台

状态：confirmed  
更新时间：2026-05-02

## 1. 章节目标

构建默认首页监控台，做到“先看态势，再进执行”，并且不干扰 Today 日常主路径。

## 2. 已确认决策

1. 刷新策略：自动轮询 + 手动刷新
2. 轮询间隔：默认 10 秒，支持 5/10/30 秒切换
3. 默认时间窗口：最近 1 小时
4. 系统健康异常：顶部固定红色告警条 + 一键跳详情
5. drill-down 跳 Today：自动带失败分组筛选 + 时间窗口
6. 首页四大卡片默认每卡展示 5 行
7. 支持静音模式（仅保留红色告警）
8. 导出能力：仅支持当前视图 CSV

## 3. 章节计划（可执行）

1. 首屏层
   - 实现四大卡片按既定优先级展示
   - 每卡默认 5 行，支持点击查看更多
2. 刷新层
   - 默认 10 秒轮询
   - 用户可切换 5/10/30 秒
   - 手动刷新按钮始终可用
3. 告警层
   - 系统异常时展示固定红色告警条
   - 告警支持跳转到对应详情
4. 联动层
   - drill-down 保留失败分组与时间窗口上下文跳转 Today
5. 导出层
   - 导出当前视图 CSV
6. 体验层
   - 静音模式屏蔽非关键提醒

## 4. 验收标准

1. 监控页默认 10 秒自动刷新且可手动刷新。
2. 用户可切换 5/10/30 秒轮询间隔。
3. 默认时间窗口为最近 1 小时。
4. 异常时顶部固定红色告警条可见且可跳详情。
5. drill-down 到 Today 时自动带上失败分组和时间窗口。
6. 四大卡片默认每卡显示 5 行。
7. 静音模式生效后仅保留红色告警。
8. 支持当前视图 CSV 导出。

## 5. 风险与回退

1. 风险：高频轮询增加资源压力。
   - 处置：默认 10 秒并允许切换到 30 秒。
2. 风险：告警过多造成疲劳。
   - 处置：静音模式屏蔽非关键提醒。
3. 风险：上下文跳转不一致。
   - 处置：统一 drill-down 参数协议并回归验证。
4. 风险：导出泄露无关数据。
   - 处置：仅导出当前过滤视图。
