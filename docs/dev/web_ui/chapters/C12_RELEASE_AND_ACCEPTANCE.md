# C12 验收与发布策略

状态：confirmed  
更新时间：2026-05-02

## 1. 章节目标

建立可持续的验收与发布流程，在保证 Today 可用性的前提下稳定迭代。

## 2. 已确认决策

1. 发布节奏：按需求不定期发布
2. 发布前门禁：自动化 + Today 手工冒烟
3. 发布失败回退：先关 feature flag，再回滚版本
4. 验收签核：你 + 验收清单双确认

## 3. 章节计划（可执行）

1. 发布准备
   - 产出发布变更清单
   - 执行自动化测试
   - 执行 Today 手工冒烟
2. 发布执行
   - 按需求窗口执行发布
   - 发布后观察关键看板与告警
3. 异常回退
   - 首先关闭相关 feature flag
   - 必要时回滚版本
4. 验收签核
   - 我提供结构化验收清单
   - 你确认通过后签核

## 4. 验收标准

1. 每次发布前自动化测试通过。
2. 每次发布前 Today 手工冒烟通过。
3. 失败场景可在最短路径执行“关开关 -> 回滚”。
4. 每次发布均有双确认签核记录。

## 5. 风险与回退

1. 风险：不定期发布导致节奏波动。
   - 处置：每次发布附带最小发布包与清单。
2. 风险：手工冒烟执行不一致。
   - 处置：固定 Today 冒烟脚本与检查项。
3. 风险：回退动作不及时。
   - 处置：预先准备开关与回滚指令模板。
