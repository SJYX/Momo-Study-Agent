# C04 TaskDrawer 执行上下文

状态：confirmed  
更新时间：2026-05-02

## 1. 章节目标

在不打断 Today 主路径的前提下，提供可随时查看的任务上下文与错误定位能力。

## 2. 已确认决策

1. 入口形态：智能 icon 模式（平时悬浮 icon，出现高风险错误自动弹开）
2. 执行开始后不强制全抽屉占用视图，以轻提示为主
3. icon 位置：固定右下角
4. icon 信息：状态点 + 数字计数
5. 自动弹开后：直接定位错误分组/错误明细区
6. 任务结束后：抽屉自动收起，仅保留 icon 提示
7. 日志保留：仅保留最近一次任务日志

## 3. 章节计划（可执行）

1. 入口层
   - 右下固定悬浮 icon
   - 状态点与计数联动任务状态
2. 弹开层
   - 高风险错误触发自动弹开
   - 默认定位错误分组/明细
3. 收口层
   - 任务终态自动收起抽屉
   - icon 保留状态提示
4. 日志层
   - 折叠日志区默认仅保留最近一次任务日志

## 4. 验收标准

1. 常态下不占据主视图，仅显示右下 icon。
2. 高风险错误发生时抽屉自动弹开并定位错误明细。
3. icon 显示状态点与计数，且状态实时更新。
4. 任务结束后抽屉自动收起，仅保留 icon。
5. 日志区仅显示最近一次任务日志。
6. 全流程通过全局 Today 门禁。

## 5. 风险与回退

1. 风险：自动弹开可能打断操作。
   - 处置：仅高风险错误触发，普通错误不强弹。
2. 风险：自动收起导致结果被忽略。
   - 处置：icon 保留计数/状态提示，支持一键重新展开。
3. 风险：日志仅保留一次导致排障信息不足。
   - 处置：详细历史交由监控台/后端日志追踪。
