# C06 Future/Iteration 语义统一

状态：confirmed  
更新时间：2026-05-02

## 1. 章节目标

将 Future 与 Iteration 的交互语义与 Today 高度统一，降低学习与操作切换成本。

## 2. 已确认决策

1. 统一程度：高统一（筛选、排序、执行、失败分组、重试行为尽量一致）
2. 默认筛选：Future/Iteration 都默认仅显示可执行项
3. 执行前提示：两页都沿用“本次将执行 N 条”的轻确认条
4. 失败分组：两页完全沿用 C03（含默认展开最大失败组、>100 条二次确认）
5. 默认排序：两页都与 Today 一致（价值优先 + 时间压力次级）
6. 执行中滚动：两页都自动滚动到当前处理行
7. 完成后行为：两页都停留在结果摘要区
8. 抽屉行为：两页都默认任务结束后自动收起，仅保留 icon
9. 静音模式：两页都支持“仅红色告警”
10. 验收优先级：语义一致性优先

## 3. 章节计划（可执行）

1. 语义层
   - 对齐状态词典与错误分组词典
   - 对齐执行前确认与完成后停留行为
2. 交互层
   - 对齐筛选默认值、排序策略、自动滚动
   - 对齐抽屉收口与静音策略
3. 重试层
   - 对齐 C03 组级重试全套规则
4. 验收层
   - 以语义一致性为第一优先进行回归

## 4. 验收标准

1. Future/Iteration 在筛选、排序、确认、重试、回退语义上与 Today 一致。
2. 两页默认仅显示可执行项，且执行前均显示轻确认条。
3. 两页均支持自动滚动、结果摘要停留、终态抽屉自动收起。
4. 两页失败分组与 C03 规则一致（含 >100 条二次确认）。
5. 两页均支持静音模式并仅保留红色告警。
6. 全流程通过全局 Today 门禁与一致性回归。

## 5. 风险与回退

1. 风险：过度统一掩盖页面差异需求。
   - 处置：保留场景特化字段，不改变语义主干。
2. 风险：大列表自动滚动影响阅读。
   - 处置：复用 C04 的跟随可控策略（实现时补开关）。
3. 风险：一致性改造引入回归。
   - 处置：按语义一致性优先的回归矩阵逐项验证。
