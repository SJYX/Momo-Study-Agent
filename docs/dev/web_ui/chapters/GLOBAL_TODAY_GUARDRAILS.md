# 全局 Today 可用性保护门禁（开发全周期）

状态：confirmed  
更新时间：2026-05-02

## 1. 目的

在整个 Web 开发周期中，确保 Today（今日单词）持续可用，不影响日常使用。

## 2. 适用范围

1. 所有 Web 开发阶段（C01-C12）
2. 任何直接或间接影响 Today 主路径的前后端改动
3. 任何涉及 Task/SSE/行级状态聚合的改动

## 3. 强制门禁（已确认：平衡档）

每次相关改动合入前，必须同时通过：

1. Today 冒烟 E2E
2. Today/Task/SSE 相关测试集
3. 前端 `build + typecheck`

说明：任一失败，禁止合入。

## 4. 开关与回退要求

1. 影响 Today 主路径的新交互必须置于 feature flag 之后
2. 默认保持当前稳定路径可用
3. 上线后出现明显回归时优先执行开关回退

## 5. 发布策略

1. 小步发布，单次变更控制范围
2. 先灰度（本地/测试用户）再全量
3. 发布后执行 Today 快速手工回归（核心 3-5 条路径）

## 6. 与章节推进关系

1. C02 不暂停，继续开发
2. C03-C12 均受本门禁约束
3. 任一章节产出若影响 Today，必须先满足门禁再合入
