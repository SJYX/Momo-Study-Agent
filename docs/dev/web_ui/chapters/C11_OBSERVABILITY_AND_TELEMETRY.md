# C11 可观测性与埋点

状态：confirmed  
更新时间：2026-05-02

## 1. 章节目标

建立可观测闭环，让效率、稳定性、回归风险都可量化追踪。

## 2. 已确认决策

1. 埋点采样：关键事件全量采集
2. 性能指标：每次页面进入都采集
3. 埋点保留时长：30 天
4. 异常通知：站内 + 邮件（后续预留）
5. 按 profile 维度提供独立看板
6. 提供 Today 可用性专属看板（成功率/中断率/回退次数）
7. 邮件通知默认触发级别：仅严重
8. 支持一键导出埋点报表 CSV

## 3. 章节计划（可执行）

1. 事件层
   - 关键事件全量埋点
   - 页面进入性能采集
2. 存储层
   - 埋点保留 30 天
3. 展示层
   - profile 维度独立看板
   - Today 可用性专属看板
4. 告警层
   - 站内告警
   - 邮件通知能力预留（仅严重）
5. 导出层
   - 埋点报表 CSV 一键导出

## 4. 验收标准

1. 关键事件与性能指标均被稳定采集。
2. 数据按 30 天策略保留。
3. 可切换 profile 查看独立看板。
4. Today 可用性看板可见且指标完整。
5. 严重级异常可触发站内告警，邮件能力预留可验证。
6. 支持一键导出当前看板相关 CSV。

## 5. 风险与回退

1. 风险：全量采集导致数据量增长。
   - 处置：保留窗口 30 天并优化存储结构。
2. 风险：告警噪音。
   - 处置：邮件仅严重级触发。
3. 风险：跨 profile 数据误读。
   - 处置：看板显式展示当前 profile 与时间窗口。
