# C10 风险控制与确认策略

状态：confirmed  
更新时间：2026-05-02

## 1. 章节目标

在效率优先前提下控制误操作与高风险行为，确保可解释、可撤销、可回退。

## 2. 已确认决策

1. 高风险弹窗文案：详细版（影响范围 + 后果 + 回退方式）
2. 高风险动作冷静期：3 秒
3. 批量操作撤销窗口：10 秒软撤销
4. 高风险操作权限：仅开启“高级模式”后可执行
5. 高级模式开启需要二次口令（本地 PIN）
6. 高级模式有效时长：30 分钟
7. 撤销倒计时提示：页面顶部横条
8. 高风险动作必须写审计日志

## 3. 章节计划（可执行）

1. 风险分级层
   - 定义高风险动作清单
   - 统一走详细确认弹窗
2. 防误触层
   - 高风险动作前 3 秒冷静期
   - 仅高级模式可操作
3. 权限会话层
   - PIN 开启高级模式
   - 30 分钟自动失效
4. 撤销层
   - 批量动作执行后提供 10 秒软撤销
   - 顶部横条倒计时提示
5. 审计层
   - 高风险动作全量记录审计日志

## 4. 验收标准

1. 高风险动作均展示详细确认信息。
2. 高风险动作执行前存在 3 秒冷静期。
3. 未开启高级模式无法执行高风险动作。
4. 高级模式需 PIN，且 30 分钟后失效。
5. 批量动作有 10 秒软撤销窗口与顶部倒计时提示。
6. 所有高风险动作写入审计日志。

## 5. 风险与回退

1. 风险：额外确认影响操作速度。
   - 处置：仅高风险动作启用重门禁。
2. 风险：PIN 遗忘造成阻断。
   - 处置：提供本地重置流程（需二次确认）。
3. 风险：撤销与后端状态不同步。
   - 处置：撤销仅限可逆动作，失败时给明确提示并落审计。
