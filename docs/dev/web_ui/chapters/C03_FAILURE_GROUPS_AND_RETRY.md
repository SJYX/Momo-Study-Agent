# C03 失败分组与组级重试

状态：confirmed  
更新时间：2026-05-02

## 1. 章节目标

把 Today 的失败项从“零散错误”变成“可分组、可解释、可重试”的稳定闭环，且不影响日常可用性。

## 2. 全局约束（继承）

1. 受 `GLOBAL_TODAY_GUARDRAILS.md` 约束。
2. 所有重试行为必须保持可回退、可追踪、可幂等。

## 3. 已确认决策

1. 失败分组默认展开：`失败最多的分组`
2. 组级重试默认范围：`该组全部失败项`
3. 组级重试默认使用轻确认条；高风险条件再弹窗
4. 组级重试后仍失败：留在当前分组并高亮仍失败项
5. 当失败组条目很多（>100）时，组级重试前必须二次弹窗确认
6. 失败分组默认排序：按失败数量降序
7. 不支持手动合并/拆分失败组，保持自动分组
8. 暂不引入“智能建议重试”按钮，先保证稳定闭环

## 4. 章节计划（可执行）

1. 分组层
   - 统一 error_type/error_code/phase 生成分组键
   - 默认按失败数量降序，自动展开第一组
2. 重试层
   - 组级默认全量重试
   - 轻确认条展示“将重试 N 条”
   - >100 条触发二次弹窗确认
3. 结果层
   - 重试后保留当前分组上下文
   - 高亮仍失败项，支持继续重试
4. 稳定性层
   - 与幂等 request_id 对齐
   - 保证重试动作可追踪和可回放

## 5. 验收标准

1. 失败项可自动正确分组，默认展开失败最多分组。
2. 分组顺序按失败数量降序且稳定。
3. 组级重试默认覆盖该组全部失败项。
4. 重试前显示轻确认条；>100 条时出现二次弹窗。
5. 重试后仍失败时保持当前分组并高亮残留失败项。
6. 不出现手动合并/拆分入口。
7. 不出现智能建议重试入口。
8. 全流程通过全局 Today 门禁。

## 6. 风险与回退

1. 风险：自动分组语义不符合少数边缘错误。
   - 处置：先保证稳定分组，不做手工编辑；后续可在 C08 扩展分组键。
2. 风险：大批量重试误触发。
   - 处置：>100 条强制二次确认。
3. 风险：重复重试导致状态噪音。
   - 处置：保留上下文并高亮“仍失败项”，避免重复盲点。
