## 3. 批次明细

## V1：Today 主闭环稳定化（C02+C03）

交付内容：

1. 行视图优先 + 默认可执行项筛选
2. 价值优先 + 时间压力次级排序
3. 执行前轻确认条
4. 执行中自动跟随当前行 + 可改筛选（仅影响显示）
5. 完成后停留摘要
6. 失败分组默认展开最大失败组
7. 组级全量重试、>100 条二次确认、残留失败高亮

验收重点：

1. Today 主路径可连续完成“执行-失败-重试”
2. 不影响现有日常任务处理
3. 回归全绿

风险控制：

1. 新交互全部 behind flag
2. 任意异常先关开关

## V2：执行上下文与监控台（C04+C05）

交付内容：

1. TaskDrawer 智能 icon 模式（右下固定）
2. 高风险错误自动弹开并定位错误明细
3. 终态自动收起，仅保留 icon
4. Ops Monitor 默认首页
5. 10 秒轮询 + 手动刷新 + 5/10/30 切换
6. 异常红条、drill-down 上下文跳转、CSV 导出

验收重点：

1. 不打断 Today 主操作
2. 监控到执行跳转链路稳定

## V3：语义统一扩展（C06+C07）

交付内容：

1. Future/Iteration 与 Today 高统一
2. 二级页（Word/Sync/Preflight）高统一
3. 静音模式一致化

验收重点：

1. 用户跨页切换无语义割裂
2. 二级页可快速定位问题

## V4：契约与效率增强（C08+C09）

交付内容：

1. 终态不可逆状态机
2. 领域错误码 + 稳定编号
3. 前端 request_id 幂等透传（30 分钟去重）
4. SSE 3 次重连超限降级手动刷新
5. 8 个核心快捷键 + 首次引导 + `?` 速查

验收重点：

1. 协议稳定不回归
2. 快捷键可用且低误触

## V5：治理收口（C10+C11+C12）

交付内容：

1. 高风险高级模式（PIN + 30 分钟）
2. 3 秒冷静期 + 10 秒软撤销
3. 审计日志强制
4. 全量关键埋点 + Today 可用性看板 + profile 看板
5. 发布验收双确认机制

验收重点：

1. 风险动作可控可追踪
2. 发布机制可复用
