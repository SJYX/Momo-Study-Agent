# DECISIONS.md — 设计决策日志

记录所有"为什么不那样做"的决策，防止未来重蹈覆辙。每次大功能迭代后更新。

---

## DEC-001 · 不干预主线复习队列

**日期**: 2026-04-10  
**场景**: 薄弱词重炼后需要强化复习  
**否定方案**: 调用 `POST /api/v1/study/advance_study` 将薄弱词强推进今日复习  
**采用方案**: 将薄弱词汇聚到私有云词本 `MomoAgent: 薄弱词攻坚`（UNPUBLISHED）  
**原因**: 用户明确表示不希望脚本影响其既定的每日打卡节奏。云词本作为独立模块，用户可自主选择时机消灭顽固词汇。

## DEC-002 · 数据库双轨策略

**日期**: 2026-04  
**场景**: 生产数据库选型  
**否定方案**: 纯云端 Turso / 纯本地 SQLite  
**采用方案**: 本地 SQLite（`data/history_*.db`）+ Turso 云端双向同步  
**原因**: 
- 网络抖动时本地继续工作，不阻塞 AI 生成流程
- 程序启动时本地数据库结构升级不依赖云端连接
- 多设备通过云端同步，本地作为热数据缓存

## DEC-003 · 禁止在云端连接上设置 row_factory

**日期**: 2026-04-10  
**场景**: SQLite 查询结果映射为字典  
**否定方案**: `conn.row_factory = sqlite3.Row`  
**采用方案**: `_row_to_dict(cursor, row)` 兼容函数  
**原因**: Turso 的 `libsql.Connection` 对象（`builtins.Connection`）不支持 `row_factory` 属性赋值，直接抛出 `AttributeError`，导致选项 3 意外崩溃。

---

## DEC-004 · Token 验证使用低门槛探针

**日期**: 2026-04-10  
**场景**: 新用户配置向导中验证 MaiMemo Token 合法性  
**否定方案**: `api.get_today_items(limit=1)` 作为验证探针  
**采用方案**: `api.get_study_progress()` 作为验证探针  
**原因**: 新用户若尚未在 App 内选定词书，`get_today_items` 会返回异常结构或空数据，导致系统误判 Token 无效，卡死初始配置流程。

---

## DEC-005 · 批量同步代替逐条请求

**日期**: 2026-04  
**场景**: 本地与云端数据库同步  
**否定方案**: 逐条记录发起网络请求  
**采用方案**: 每 100 条记录为一批（Chunk）发起一次请求  
**原因**: 逐条请求在数据量大时产生"假死"感（数千次网络往返），批量化后显著降低延迟，消除用户等待压力。

---

## DEC-006 · Prompt 文件统一归入 docs/prompts/

**日期**: 2026-04-10  
**场景**: Prompt 文件管理  
**否定方案**: Prompt 文件散落在根目录  
**采用方案**: 统一归入 `docs/prompts/`，路径通过 `config.py` 中的常量引用  
**原因**: 随项目增长，根目录文件夹混乱，影响导航效率。Prompt 文件属于文档资产，归入 docs/ 体系更合理。
