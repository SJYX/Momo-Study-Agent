# MoMo Web UI 交互路径全量清单（A+）

更新时间：2026-05-02  
关联计划：`docs/dev/WEB_UI_EXEC_PLAN.md`（A+ 重构版）  
文档目标：提供可直接进入研发拆解的交互路径基线（页面级）

---

## 0. 范围与说明

1. 本文聚焦交互路径，不展开视觉稿细节。
2. 每条路径包含：入口、前置条件、主流程、异常分支、回退路径、出口。
3. 全站采用统一状态语义：
   - task：`pending/running/done/error/canceled`
   - row：`pending/running/done/error/skipped/sync_pending/sync_conflict/sync_failed`

---

## 1. 全局路径规则

## 1.1 全局入口

1. 首次访问：`UserGateway`
2. 已有 active profile：默认直达 `Ops Monitor`
3. 非法路由：`NotFound -> Today 或 UserGateway`

## 1.2 全局守卫

1. 无 active profile：业务页统一重定向 `UserGateway`
2. profile 切换时：
   - 关闭旧 SSE
   - 清理旧 task 上下文
   - 刷新当前页数据
3. 高风险动作：统一二次确认

## 1.3 全局回退

1. 任何业务页异常可回退到 `Today`
2. 网络异常统一显示可重试 Banner
3. SSE 中断自动重连，超过阈值提示手动刷新

---

## 2. UserGateway 交互路径

## 2.1 主路径：选择已有 profile

1. 入口：首次进入或被路由守卫拦截
2. 前置：`GET /api/users` 成功
3. 流程：选择 profile -> 写入 localStorage -> 跳转 Today
4. 出口：Today 加载对应 profile 数据

## 2.2 次路径：创建 profile

1. 入口：UserGateway 新建表单
2. 前置：profile_name 合法
3. 流程：提交 `POST /api/users` -> 成功后自动选中 -> 跳转 Today
4. 出口：Today 首次空数据引导

## 2.3 异常路径

1. users 拉取失败：页面内错误 + 重试按钮
2. 创建冲突/非法命名：字段级报错
3. 创建成功但 session 不一致：toast + 强制刷新 session

## 2.4 回退路径

1. 可取消创建返回列表
2. API 连续失败时保留离线提示，不自动跳页

---

## 3. Today Command Center（核心）

## 3.1 主路径：批处理闭环

1. 入口：从 Ops Monitor 点击“进入 Today 执行”或侧边栏进入
2. 前置：profile 已激活，Today 数据加载成功
3. 流程：
   - 设置筛选/排序（默认价值优先）
   - 选择批次
   - 执行任务
   - 实时查看总进度与行级状态
   - 任务终态后查看结果摘要
4. 出口：done 或 error（进入失败分组路径）

## 3.2 主路径：失败分组与组级重试

1. 入口：任务终态为 error 或存在失败行
2. 前置：失败事件具备 `error_type/error_code/phase`
3. 流程：
   - 自动按失败类型分组
   - 展开某组看组内条目
   - 选择“组级重试”或“排除后重试”
   - 触发重试任务并追踪新进度
4. 出口：
   - 重试成功 -> 完成
   - 仍失败 -> 留在分组面板并可继续细分

## 3.3 次路径：单条处理

1. 行内操作：查看详情 -> 单条重试/跳过
2. 适用：组内少量异常、不适合整组重试

## 3.4 次路径：任务抽屉（TaskDrawer）

1. 入口：点击任务摘要或快捷键 `D`
2. 流程：查看阶段、进度、统计、折叠日志
3. 出口：关闭抽屉返回列表上下文

## 3.5 异常路径

1. 执行时 409（锁冲突）：提示“同 profile 正在运行”并给跳转监控台入口
2. SSE 中断：状态条提示重连中，超时后允许手动恢复
3. 数据刷新失败：保留最后快照 + 重试
4. 总进度与行状态不一致：标记为一致性异常并上报

## 3.6 回退路径

1. 执行中允许最小化任务上下文，返回列表继续查看
2. 中止任务后进入 canceled 终态，允许重新发起
3. 用户离开页面再返回，自动恢复最近任务视图

---

## 4. Future 页面路径

## 4.1 主路径

1. 入口：侧边栏 Future
2. 前置：查询参数有效
3. 流程：加载列表 -> 触发任务 -> 行级进度 -> 结果回写
4. 出口：done/error

## 4.2 与 Today 对齐项

1. 同状态词典
2. 同失败分组策略
3. 同组级重试能力

## 4.3 异常与回退

1. 查询为空：空态引导可跳转 Today
2. 参数非法：前置校验阻断请求

---

## 5. Iteration 页面路径

## 5.1 主路径

1. 入口：侧边栏 Iteration
2. 流程：候选列表 -> 批处理执行 -> 行级进度 -> 完成摘要
3. 出口：done/error

## 5.2 次路径

1. 失败项定位后可跳回 Today 做补处理
2. 支持按候选分组进行局部重试

## 5.3 异常与回退

1. 候选为空：展示原因与刷新入口
2. 批次失败：保留成功项结果，不全量回滚

---

## 6. Ops Monitor（独立监控台）

## 6.1 主路径：全局态势查看

1. 入口：导航 Ops Monitor
2. 流程：
   - 查看运行任务总览
   - 查看失败热区
   - 查看队列深度/延迟
   - 查看系统健康信号
3. 出口：保持只读，按需 drill-down

## 6.2 主路径：drill-down 跳转

1. 入口：点击某任务/失败组
2. 流程：定位到对应业务页 + 保留过滤上下文
3. 出口：进入 Today/Future/Iteration 的具体上下文

## 6.3 异常与回退

1. 监控数据源不可用：显示降级状态，不阻断业务页
2. 时间窗口无数据：展示“无事件”空态

---

## 7. Word Library 路径

## 7.1 主路径

1. 入口：导航 Word Library
2. 流程：分页/搜索 -> 查看详情 -> 查看迭代历史
3. 出口：返回列表并保留位置

## 7.2 异常与回退

1. 搜索无结果：空态 + 清空筛选
2. 详情加载失败：局部重试，不清空列表

---

## 8. Sync Status 路径

## 8.1 主路径

1. 入口：导航 Sync Status
2. 流程：查看队列/失败摘要 -> 手动 flush -> 查看结果
3. 出口：状态更新并可跳转到失败上下文

## 8.2 异常与回退

1. flush 失败：保留失败项并给 fix hint
2. 冲突增多：引导进入 Ops Monitor 热区视图

---

## 9. Preflight 路径

## 9.1 主路径

1. 入口：导航 Preflight
2. 流程：执行健康检查 -> 展示 pass/warn/fail -> 提示修复建议
3. 出口：可一键重跑

## 9.2 异常与回退

1. 检查超时：可中止并重跑
2. 连续失败：导向 Ops Monitor / 日志抽屉定位

---

## 10. Settings（含 profile 切换）路径

## 10.1 主路径

1. 入口：导航 Settings
2. 流程：修改配置 -> 保存 -> 回写 session 快照
3. 出口：回到来源页

## 10.2 profile 切换路径

1. 入口：顶部 profile 切换器
2. 流程：切换 -> 清理上下文 -> 重建订阅 -> 当前页刷新
3. 出口：新 profile 对应数据

## 10.3 异常与回退

1. 切换失败：恢复旧 profile + 错误提示
2. 部分配置无效：字段级阻断保存

---

## 11. 快捷键路径（中等强度）

1. `F`：打开筛选
2. `Ctrl+Enter`：执行当前批次
3. `G`：打开失败分组
4. `R`：组级重试
5. `Ctrl+K`：全局搜索
6. `S`：切换排序
7. `D`：打开任务抽屉
8. `Shift+R`：刷新状态

约束：输入框聚焦时仅保留安全快捷键，避免误触发执行。

---

## 12. 高风险动作确认矩阵

必须二次确认：

1. 批量覆盖式重跑
2. 可能写入持久化且不可逆动作
3. 跨大量条目的强制重试

确认弹窗必须展示：

1. 影响条目数
2. 影响范围（profile/页面）
3. 可撤回性说明
