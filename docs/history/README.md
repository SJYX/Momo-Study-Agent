# 历史文档索引

本目录存放不再作为当前实现依据的历史资料。它的作用是把“已经完成的方案”分层归档，避免活动文档和历史快照混在一起。

## 子目录

- `phases/`：历史阶段方案、完成记录、路线复盘
- `snapshots/`：时间点快照、一次性审查、阶段总结
- `web_ui_legacy/`：旧版 Web UI 方案与交互路径

## 使用建议

- 想看“现在怎么做”时，优先阅读 `docs/dev/` 与 `docs/architecture/`。
- 想看“为什么当时这么做”时，查 `docs/dev/DECISIONS.md`、`docs/dev/SYNC_OPTIMIZATION_PLAYBOOK.md` 的历史层，再按索引进入 `history/`。
- 想看“某阶段已经完成了什么”时，去 `history/phases/`。
- 想看“某个时间点系统长什么样”时，去 `history/snapshots/`。
