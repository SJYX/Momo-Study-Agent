# Today Tasks 交互逻辑优化设计 (Hybrid Smart Dashboard)

## 0. 背景与目标
当前 Today Tasks 页面的交互逻辑主要基于简单的状态流转，缺乏对大批量任务的精细筛选能力，且在运行监控和错误排查时的交互较为基础。
本项目旨在采用「混合智能工作台」理念，结合统一的暖色 Notion 设计语言，全面提升任务启动、执行监控、异常反馈及批量操作的效率。

## 1. 架构与交互模型
采用 **方案 C: 混合智能工作台 (Hybrid Smart Dashboard)**。
核心理念：**默认状态下保持极简和傻瓜式的一键操作，通过强大的视图过滤器驱动批量任务；在需要时，允许用户进入类似移动端相册的批量选择模式进行精细控制。**

## 2. 核心模块设计

### 2.1 增强型 Hero 区与视图驱动
Hero 区从单纯的数据展示转变为**状态感知**的控制中枢。

*   **数据概览**：左侧展示环形进度条 (Ring Chart) 和大号完成度统计（如 `85/120 进度 70%`）。
*   **强力过滤器 (Filter Bar)**：
    *   **快速视图 Pill**：`[全部 (120)]` `[待处理 (20)]` `[已失败 (4)]` `[新词 (15)]` `[复习词 (105)]`。
    *   **高级筛选组合**：提供 `[ ⚙️ 更多筛选 ]` 下拉菜单，支持按阶段 (如 API 失败、AI 失败) 或错误类型筛选。
*   **智能主操作 (Context-Aware CTA)**：
    *   右侧常驻固定的主操作按钮。
    *   **视图绑定**：主操作按钮的文案与行为直接绑定当前激活的过滤器视图。例如，在“新词”视图下，按钮显示为 `[ 🚀 处理 15 个新词 ]`。
    *   **次级快捷按钮**：当存在失败任务时，主按钮旁会显示红色次级按钮 `[ ↻ 重试 N 个失败 ]`，提供一键恢复能力。

### 2.2 行内状态监控增强 (Inline Monitoring)
在任务执行期间，为单行提供更具生命力的视觉反馈。

*   **高亮与呼吸动画**：当前正在执行 (Running) 的行背景变为淡色高亮（`bg-surface-highlight`），并带有轻微的呼吸闪烁 (Pulse) 动画，引导视觉焦点。
*   **状态 Pill 动画**：行内的状态标签图标变为旋转加载态 (Spinner)，并实时更新子阶段文字。
*   **智能自动跟随**：列表默认自动滚动以保持高亮行在视口中央。一旦检测到用户的手动滚动行为，自动跟随立刻暂停，并在界面提示 `[恢复跟随]`。

### 2.3 异常排查与原位重试 (Inline Error Handling)
摒弃脱离上下文的独立错误面板，强化原位排查能力。

*   **视觉阻断**：失败行背景变为浅红色 (`bg-error-soft`)，立刻从默认的灰白列表中凸显。
*   **原位快捷重试**：失败行的最右侧直接显示红色的幽灵按钮 (Ghost Button) `[ ↻ 单独重试 ]`。点击后该行直接转入 Running 状态。
*   **直观错误原因**：原状态栏位置截断显示具体错误原因（如 "网络超时"），鼠标悬浮 (Hover) 可通过 Tooltip 查看完整的后端异常堆栈，便于排查诊断。

### 2.4 浮动批量控制流 (Floating Batch Control)
处理需要精确挑选（如挑出 5 个词）的边缘场景。

*   **按需开启**：表格默认隐藏 Checkbox，保持界面清爽。用户点击右上角 `[ 🔲 进入批量选择 ]` 后，表格行整体微移，滑出 Checkbox 列。
*   **快捷交互**：支持 Shift + Click 连选。
*   **浮动底栏 (Bottom Bar)**：
    *   进入批量模式后，视口底部固定升起深色/高亮操作栏。
    *   显示选中统计：`已选择 X 个词`。
    *   提供操作组：左侧 `[ 取消选择 ]`，右侧 `[ 🚀 处理选中项 ]` 或其他批量动作。
*   **用完即走**：一旦执行动作或取消，Checkbox 隐藏，Bottom Bar 降下，恢复极简常态。

## 3. 实施路径 (Implementation Strategy)
建议按以下步骤逐步实现：
1.  **Phase 1: Filter & Hero** - 重构 `TodayHero` 组件结构，引入视图枚举、环形进度图，实现过滤器联动和智能按钮动态文案。
2.  **Phase 2: Inline Status & Error** - 更新 `TodayTasksV2` 中表格行的渲染逻辑。增加针对 `status === 'running'` 的背景闪烁动画样式类，添加 `status === 'error'` 时的重试小按钮与 tooltip 逻辑。
3.  **Phase 3: Batch Mode & Bottom Bar** - 引入多选状态（如 `selectedIds` set），编写悬浮 `BottomBar` 组件，在批量模式激活时展示并处理勾选项逻辑。
